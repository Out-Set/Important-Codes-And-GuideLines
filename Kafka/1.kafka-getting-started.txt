Kafka:
------
Kafka 0.8 -> 2.x :: ZooKeeper required
Kafka 3.0 -> 3.3 :: ZooKeeper required (KRaft optional, NOT production ready)
Kafka 3.4 -> 3.9 :: Both ZooKeeper and KRaft supported (KRaft stable)
Kafka 4.0+ :: ZooKeeper removed - KRaft mandatory

Documentation: https://kafka.apache.org/documentation/

Note:
-----
You can use ZooKeeper until Kafka 3.9.x
Starting from 4.0 only KRaft mode is supported -- no more ZooKeeper.

Download Kafka binary then extarct it: https://kafka.apache.org/downloads

Start Zookeeper Server: ./bin/zookeeper-server-start.sh ./config/zookeeper.properties
Start Kafka Server: ./bin/kafka-server-start.sh ./config/server.properties

Create a Topic: ./bin/kafka-topics.sh --bootstrap-server localhost:9092 --create --topic myabc
Since you haven't given any partition information, there will be only 1 partition myabc-0 by default
Note: You can change the default settings of partition creation, going to the /config/server.properties (num.partitions=1) to any number.

Create a Topic with partition information: ./bin/kafka-topics.sh --bootstrap-server localhost:9092 --create --topic mypqr --partitions 3
Now there will be 3 partitions mypqr-0, mypqr-1 and mypqr-2

Note: To view these topic open /tmp/kafka-logs dir by running command: xdg-open /tmp/kafka-logs

Get the description of a topic: ./bin/kafka-topics.sh --bootstrap-server localhost:9092 --topic mypqr --describe

Open Produccer Console: ./bin/kafka-console-producer.sh --bootstrap-server localhost:9092 --topic mypqr
Open Consumer Console & consume on real time: ./bin/kafka-console-consumer.sh --bootstrap-server localhost:9092 --topic mypqr
Open Consumer Console & consume everything from beginning: ./bin/kafka-console-consumer.sh --bootstrap-server localhost:9092 --topic mypqr --from-beginning

Consume messages with Partition, key and value information: --property print.partition=true --property print.value=true
./bin/kafka-console-consumer.sh --bootstrap-server localhost:9092 --topic mypqr --property print.partition=true --property print.key=true --property print.value=true --from-beginning
